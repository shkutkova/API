# Проект API-тестирования

Это проект по автоматизированному тестированию API на Python с использованием pytest и Allure для отчетов. Проект загружен на GitHub: [https://github.com/shkutkova/API](https://github.com/shkutkova/API).

## Структура проекта

```
API/
├─ .idea/                  # Настройки IDE (например, PyCharm)
├─ HW/                     # Домашние задания / примеры
├─ __pycache__/            # Автогенерируемые файлы Python (.pyc)
├─ allure_results/         # Результаты тестов для Allure
├─ data/                   # Тестовые данные (JSON)
├─ modules/                # Основной код проекта, который тестируем или вспомогательные модули
├─ old/                    # Старый код, черновики
├─ tests/                  # Файлы с автотестами
├─ utils/                  # Вспомогательные функции и утилиты
├─ .env                    # Переменные окружения (URL API, токены)
├─ conftest.py             # Фикстуры для pytest (setup/teardown, подготовка клиента)
├─ logs.log                # Логи выполнения тестов
├─ pytest.ini              # Конфигурация pytest (марки, настройки, репорты)
├─ requirements.txt        # Список зависимостей
```

## Подробное описание папок

### modules/

Содержит основной код проекта:

* API-клиенты (отправка запросов, получение ответов)
* Модели данных
* Вспомогательные функции для работы с API

### utils/

Вспомогательные функции, которые используют тесты и модули:

* Логирование
* Генерация и подготовка данных
* Проверка ответов API
* Чтение переменных окружения

Например: функция, которая читает данные из .env, или общая функция для «отправить запрос + проверить статус».

### data/

Тестовые данные для параметризованных тестов:

* JSON, CSV, YAML
* Шаблоны данных для различных сценариев

### tests/

Автотесты проекта:

* Файлы, начинающиеся с `test_` проверяют функционал API
* Используют фикстуры (предусловия) из `conftest.py`
* Используют модули из `modules/`
* Используют данные из `data/`
* Запускаются через pytest

### allure_results/

Результаты тестов для Allure Framework:

* JSON, скриншоты, текстовые файлы
* Используется для генерации визуальных отчетов: `allure serve` или `allure generate`

### HW/ и old/

* HW — домашние задания / примеры
* old — старый код / черновики


## Файлы

### conftest.py

Файл с фикстурами для pytest:

* Настройка базового URL API
* Авторизация / создание клиента
* Setup и teardown перед/после тестов
* Фикстуры доступны во всех тестах

### .env

Файл с переменными окружения:

* URL сервера
* Логины, пароли, токены

### pytest.ini

Конфигурация pytest:

* Папка с тестами
* Маркеры
* Исключения (игнорировать ли какие‑то предупреждения)
* Настройки репортов allure
Это файл, который задаёт правила работы тестового фреймворка.

### requirements.txt

Список библиотек (например: requests, pytest, allure‑pytest, pytest‑html, jsonschema и др). 
Перед запуском проекта нужно установить их: pip install -r requirements.txt.

### logs.log

Файл с логами выполнения тестов



## Как запустить проект

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Настроить `.env` файл с необходимыми переменными окружения
3. Запустить тесты:

```bash
pytest --alluredir=allure_results
```

## Как читать проект

1. **conftest.py** — смотреть фикстуры, что они делают
2. **modules/** — понять, как отправляются запросы и обрабатываются ответы
3. **tests/** — изучать, как тесты используют фикстуры, модули и данные
4. **utils/** — вспомогательные функции для упрощения кода
5. **data/** — наборы тестовых данных
6. **allure_results/** — анализировать результаты тестов
